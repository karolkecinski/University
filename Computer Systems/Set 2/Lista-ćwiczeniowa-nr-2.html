Czy poniższe wyrażenia zawsze obliczą się do prawdy dla dwóch dowolnych wartości zmiennych x i y typu int32_t? Jeśli nie to podaj wartości, które prowadzą do obliczenia fałszu.

  1. (x > 0) || (x - 1 < 0)
Nie. Dla x = -2^31 wyrażenie (x > 0) jest fałszywe. Natomiast x - 1 = -2^31 -1 = (int32_t) 2^31 - 1, zatem również nie spełnia zdania (x - 1 < 0).
  1. (x & 7) != 7 || (x << 29 < 0)
To się zgadza. Pierwszy warunek nie jest spełniony, jedynie gdy na ostatnich trzech pozycjach w zapisie binarnym x ma jedynki, wtedy operacja (x << 29) zwróci 0b111000...0, zatem w reprezentacji int32_t liczbę ujemną, spełniającą drugi warunek.
  1. (x * x) >= 0
Nie. Dla x z przedziału otwartego (2^15.5, 2^16) badane wyrażenie nie będzie spełnione. Jest tak ponieważ, maksymalny zakres typu int32_t dla liczb z tego przedziału zostanie przekroczony tylko raz. Dla dolnej granicy przedziału otrzymamy liczbę bliską INT32_T_MIN, dla górnej granicy liczbę 0.

Zadanie 6

Piotr Witkowski Zadanie rozwiązuje p. Agnieszka Jonak.

Uzupełnij ciało funkcji zadeklarowanej poniżej.

/* Jeśli x zawiera nieparzystą liczbę jedynek zwróć 1, w p.p. 0 */
 int32_t odd_ones(uint32_t x);

Na początku sprawdzamy czy para bitów zawiera parzystą czy nieparzystą liczbę jedynek, czyli przesuwamy naszą liczbę x o jeden bit w prawo i porównujemy bity w orginalnym x i w przesuniętym.
Uzyskujemy 1 dla nieparzystej liczby i 0 dla parzystej (tak działa operator ^). Zapisujemy te informację w prawym bicie.
Później porównujemy tak pary stojące obok siebie (prawe bity par), następnie 4, później 8 i na końcu 16.
Możemy zauważyć, że interesujące nas bity, które mówią o parzystości przesuwają się coraz bardziej na prawo, zatem po ostatniej operacji bit informujący nas o parzystości jedynek w liczbie będzie ostatni.

int32_t odd_ones(uint32_t x) {
    x = (x >> 1) ^ x;
    x = (x >> 2) ^ x;
    x = (x >> 4) ^ x;
    x = (x >> 8) ^ x;
    x = (x >> 16) ^ x;
    return x & 1;
}

Załóżmy, że zmienne x, f i d są odpowiednio typów int32_t, float i double. Ich wartości są dowolne, ale f i d nie mogą równać się +∞+∞, −∞−∞ lub NaN. Czy każde z poniższych wyrażeń zostanie obliczone do~prawdy? Jeśli nie to podaj wartości zmiennych, dla których wyrażenie zostanie obliczone do~fałszu.

  1. x == (int32_t)(double) x
  2. x == (int32_t)(float) x
  3. d == (double)(float) d
  4. f == (float)(double) f
  5. f == -(-f)
  6. 1.0 / 2 == 1 / 2.0
  7. d * d >= 0.0
  8. (f + d) - f == d

Rozwiązania:

  1. True
  2. False, błąd konwersji typów (float ma mniejszą precyzję niż int32_t więc potrzeba double; przykład -> 16777217 != (int32_t)(float)16777217;
    (int32_t)(float)16777217=16777216)
  3. False, float nie pomieści double; (double)(float)16777217.000000=16777216.000000)
  4. True
  5. True
  6. True
  7. Tak, aczkolwiek może to zostać przepełnione do +infinity
  8. Nie, na przykład gdy f jakąś bardzo dużą liczbą np. 1e19, a d = 1.0 to f+d byłoby zaokrąglone 1e19 więc wtedy (f+d)-f = 0 ale po drugiej stronie mielibyśmy 1 więc fałsz
