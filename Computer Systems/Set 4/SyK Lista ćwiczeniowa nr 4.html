Rozwiązania (30 minut)

Na początku zajęć prowadzący przypisuje studentów do zadań. W ciągu 30 minut student ma obowiązek uzupełnić rozwiązanie zadania. Jeśli zadanie wymaga posłużenia się diagramem, to można wkleić jego zdjęcie. Prowadzący może zlecić przygotowanie rozwiązania danego zadania więcej niż jednej osobie.

Zamieszczanie zdjęć gotowych rozwiązań jest niedozwolone chyba, że prowadzący na to pozwoli.

Udostępnianie gotowych rozwiązań studentom z innych grup ćwiczeniowych jest niedozwolone!

Zadanie 1

Piotr Witkowski Zadanie rozwiązuje p. Patryk Wilusz.

Rejestry %reg1%%reg1% i %reg2%reg2 są tego samego rozmiaru. Wykaż, że niezależnie od zapisanych w nich wartości, interpretowanych jako liczby ze znakiem, instrukcja cmp %reg1, %reg2cmp %reg1, %reg2 ustawia flagi tak, że setl %reg3setl %reg3 zadziała zgodnie z oczekiwaniami. Podobnie, wykaż, że jeśli te wartości intepretujemy jako liczby bez znaku, to setb %reg3setb %reg3 zadziała zgodnie z oczekiwaniami. Wywnioskuj stąd, że pozostałe instrukcje rodziny setset działają stosownie do swoich sufiksów.

cmp - ustawia flagi, setl - ustaw 1 jeśli mniejsze

OF - overflow (signed)
SF - Sign Flag (a - b) < 0

setl - condition code: SF^0F
cmp %reg1, %reg2 (%reg2 - %reg1)

Niech a = %reg2, b = %reg1

Dla liczb ze znakiem:
Przypadek 1:
SF = 1, OF = 0
a - b < 0 i nie ma przepełnienia

czyli SF^OF = 1

Przypadek 2:
SF = 0, OF = 1
a - małe (ujemne)
b - duże (dodatnie)
więc a - b > 0, overflow, ale a jest mniejsze niż b

czyli SF^OF = 1

Przypadek 3:
SF = 1, OF = 1
a - duże
b - małe
więc a - b < 0 -> overflow, ale samo a nie jest mniejsze niż b

czyli SF^OF = 0

Przypadek 4
SF = 0, OF = 0
a jest większe niż b, ale nie ma przepełnienia

czyli SF^OF = 0

Dla liczb bez znaku:
setb - condition code: CF

Porównujemy liczby bez znaku, a porównanie wykonujemy a - b, więc występuje borrow, kiedy b > a, dlatego wystarczy spojrzeć na flagę CF, jeśli borrow wystąpiło wówczas a - b < 0

Dlaczego rodzina set działa? Każda instrukcja z tej rodziny posiada swoje 
condition code, które na podstawie flag decyduje czy warunek jest spełniony,
czy nie (codndition code jest zrealizowane w taki sposób, aby uwzględniać wszystkie przypadki).

Piotr Witkowski Wysokopoziomowy kod:
“Jeśli a < b to c := 1” tłumaczy się na asembler tak:

cmp b, a
setl c

Podobnie zamiast c:=1 może być instrukcja skoku, czyli “Jeśli a < b to skocz pod adres L1”

cmp b, a
jmpl L1

Zadanie 2

Piotr Witkowski Zadanie rozwiązuje p. Agata Kasprzak.

Poniżej znajduje się kod funkcji o sygnaturze void who(int16_t v[], size_t n).
Przetłumacz go na język C i odpowiedz, jaki jest efekt jego wykonania. Czy znajomość sygnatury jest istotna?

who:    subq    $1, %rsi            //n=n-1
        movl    $0, %eax            //i=0
.L2:    cmpq    %rsi, %rax          // CF=1 jeśli n>i
        jnb     .L4                 // skok jeśli CF=0 
        leaq    (%rdi,%rax,2), %rcx //
        movzwl  (%rcx), %r8d        //a = v[i]
        leaq    (%rdi,%rsi,2), %rdx //
        movzwl  (%rdx), %r9d        //b = v[n]
        movw    %r9w, (%rcx)        //v[i] = b
        movw    %r8w, (%rdx)        //v[n] = a
        addq    $1, %rax            //i++
        subq    $1, %rsi            //n--
        jmp     .L2
.L4:    ret








void who(short v[], size_t n){
    for (int i=0, j=n-1; i<j; i++, j--) {
    short pom=v[i];
    v[i]=v[j];
    v[j]=pom;
    }
}

Zadanie 3

Piotr Witkowski Zadanie rozwiązuje p. Paweł Bajgrowicz.

Poniżej znajduje się kod funkcji o sygnaturze bool zonk(char* a, char* b), jako argumenty przyjmującej C-owe łańcuchy znaków. Przetłumacz ją na język C (bez instrukcji gotogoto). Jaką wartość liczy ta funkcja?















zonk:   movl    $0, %ecx          // definiujemy i = 0
.L2:    movslq  %ecx, %rax        // podstawiam wartość %ecx na %rax, tzn. używam %rax jako i
        movzbl  (%rdi,%rax), %edx // %edx = a[i], ponieważ %rdi to wskaźnik na początek tablicy a, a %rax to przesunięcie w tej tablicy, czyli a[0 + i] i rozszerzamy %edx o 4 bajty
        testb   %dl, %dl          // testb to logiczny AND; %dl = a[i], więc jeśli a[i] = NULL, to ustawiamy flagę ZF
        je      .L6               // Skok do L6, jeśli jest ustawiona flaga ZF
        cmpb    %dl, (%rsi,%rax)  // jeśli a[i] oraz b[i] są różne, to ustawia flagę ~ZF
        jne     .L5               // Skok do L5, jeśli jest ustawiona flaga ~ZF
        addl    $1, %ecx          // i++
        jmp     .L2               // powrót na początek pętli
.L6:    orb     (%rsi,%rax), %dl  // OR logiczny b[i] z %dl(które ma wartość NULL)
        sete    %al               // Ustawia wartość %al, gdzie %al to wynik fcji, jako jeden jeśli powyższa operacja ustawiła flagę ZF, czyli i wskazuje w obydwu tablicach na NULL, więc obydwie tablice są równe
        ret                       // zwróć wartość&nbsp;%rax jako wynik fcji
.L5:    movl    $0, %eax          // ustaw wartość wyniku jako 0
        ret                       // zwróć wartość&nbsp;%rax jako wynik fcji

Funkcja sprawdza czy dwie tablice typu char są równe.











bool zonk(char* a, char* b) {
    int i = 0;
    for(; a[i]; i++) {
        if(b[i] != a[i]) {
            return 0;
        }
    }
    return (b[i] == NULL);
}

Zadanie 4

Piotr Witkowski Zadanie rozwiązuje p. Daniel Wiczołek.

Zastąp instrukcje pushq %reg1pushq %reg1 oraz popq %reg2popq %reg2 równoważnymi ciągami instrukcji jawnie operującymi na stosie.

pushq %reg1:
    sub $8, %rsp
    movq %reg1, (%rsp)

popq %reg2:
    movq (%rsp), %reg2
    addq $8, %rsp

> [PWit] addq $8, %rsp
>        movq -8(%rsp), %reg2
> alternatywnie (ale po co?)

>zamiast add %8, %rsp można napisać leaq 8(%rsp), %rsp

Odjęcie stałej od %rsp powoduje zmniejszenie wartości adresu
przechowywanego przez %rsp o tą stałą (stałą liczbę bajtów).
Następnie wrzucamy do tego adresu wartość %reg1 czyli dokładnie
8 bajtów w nim przechowywanych.

W przypadku pop najpierw zapisujemy wartość do destination.
Nastepnie zwalniamy pamięć poprzez dodanie do wskaznika stałej - 
zmniejszenie adresu wskazywanego przez %rsp o stałą liczbę bajtów.

Zadanie 5

Piotr Witkowski Zadanie rozwiązuje p. Karol Kęciński.

Poniżej znajduje się kod funkcji o sygnaturze foo(int16_t v[], size_t n). Przetłumacz ją na język C. Narysuj ramkę stosu tej funkcji i wytłumacz, jaka jest rola poszczególnych komórek ramki oraz jak jej zawartość zmienia się w trakcie działania. Jaki jest efekt ma ten kod? Jaka jest rola rejestru %rbp%rbp w tym kodzie?

foo:    pushq   %rbp
        movq    %rsp, %rbp
        movq    %rdi, -24(%rbp)
        movq    %rsi, -32(%rbp)
        movq    $0, -8(%rbp)
        jmp     .L2
.L3:    movq    -8(%rbp), %rax
        leaq    (%rax,%rax), %rdx
        movq    -24(%rbp), %rax
        addq    %rdx, %rax
        movzwl  (%rax), %eax
        leal    (%rax,%rax), %edx
        movq    -8(%rbp), %rax
        leaq    (%rax,%rax), %rcx
        movq    -24(%rbp), %rax
        addq    %rcx, %rax
        movw    %dx, (%rax)
        addq    $1, -8(%rbp)
.L2:    movq    -8(%rbp), %rax
        cmpq    -32(%rbp), %rax
        jb      .L3
        nop
        popq    %rbp
        ret


foo(int16_t v[], size_t n)
{
    rax = 0;
    
    for(int i = 0; i < n; i++)
    {
        rax = i;
        rdx = 2 * rax; // leaq (%rax,%rax), %rdx
        rax = v;    //wskaźnik na v
        rax = rax + rdx; //&v[i] (bo v to &v[0] + 2 * i, gdy typ(v)=int16_t)
        rax = *rax; //*(&v[i]) ==> v[i]
        edx = 2 * rax; //2 * v[i]
        rax = i;
        rcx = 2 * rax; // 2*i
        rax = v;
        rax = rax + rcx; //v + 2 * i = &v[i]
        *rax = edx; //v[i] = 2 * v[i]
    }
}



void foo(int16_t v[], size_t n)
{
    for(size_t i = 0; i < n; i++)
    {
        v[i] = 2*v[i];
    }
}

Zadanie 6

Piotr Witkowski Zadanie rozwiązuje p. Antoni Dąbrowski.

Poniżej znajduje się kod funkcji rekurencyjnej o nieznanej sygnaturze. Przetłumacz tę funkcję na język C, odkryj jej sygnaturę i odpowiedz, jaką wartość ona liczy.

















reccur:                           //Zmienna wejściowa jest trzymana w rejestrze %rdi
        pushq   %rbp              //Wkłada element do stosu
        movq    %rsp, %rbp        //Przypisuje rejestrowi %rbp wartość %rsp
        subq    $16, %rsp         //Zmniejsza o 16 wartość %rsp
        movl    %edi, -4(%rbp)    //Przypisuje wartość %edi pod rejestr -4(%rbp)
        cmpl    $0, -4(%rbp)      //Odejmuje od wartości pod adresem -4(%rbp) 0, zapala flagi
        jne     .L2               //Skok gdy ZF=0, czyli kiedy -4(%rbp) jest różny od zera
        movl    $1, %eax          //Przypisuje rejestrowi %eax wartość 1
        jmp     .L3               //Skok bezwarunkowy
.L2:    movl    -4(%rbp), %eax    //Przypisuje rejestrowi %eax wartość pod rejestrem -4(%rbp)
        subl    $1, %eax          //Dekrementuje %eax
        movl    %eax, %edi        //Przenosi wartość %eax pod %edi
        call    reccur            //Wykonuje ponownie funkcję reccur dla zdekrementowanej wartości
        imull   -4(%rbp), %eax    //Wymnaża wyniki
.L3:    leave                     //Podstawia %rbp pod %rsp oraz wykonuje popq %rbp 
        ret

Funkcja reccur w języku C:







int reccur(int x)
{
    if(x == 0)
        return 1;
    return x * reccur(x-1);
}

Sygnatura: int reccur(int x)
Funkcja oblicza silnię.

Zadanie 7

Piotr Witkowski Zadanie rozwiązuje p. Jakub Mendyk.

Dana jest funkcja o sygnaturze postaci int32_t bar(int32_t a1,...,int32_t an), gdzie nn jest nieznane. Jaka jest minimalna wartość nn, jeżeli wiadomo, że funkcja zwraca wartość jednego ze swoich argumentów, a jej kod wygląda tak

bar:    pushq   %rbp
        movq    %rsp, %rbp
           .....
        movl    16(%rbp), %eax
        popq    %rbp
        ret

Napisz szkic kodu asemblerowego wywołującego funkcję barbar z liczbą parametrów równą takiemu minimalnemu nn. Zadbaj o poprawne przekazanie argumentów do funkcji. Jak zmieni się napisany przez Ciebie kod, gdy nn będzie większe?

Rozwiązanie

Przeanalizujmy jak wygląda wierzchnia część stosu w momencie
rozpoczęcia wykonywania kodu asemblerowego odpowiadającego
procedurze bar:

Adres Wartość Komentarz
A + 16 zzz argument dla bar
A + 12 yyy argument dla bar albo przestrzeń by %rsp dzieliło się przez 16
A+8 aaa adres powrotu do wołającego bar
A %rbp stary rbp

Skoro wiemy, że bar zwraca wartość jednego ze swoich argumentów oraz z analizy końcówki kodu wiemy
że wartość na stosie z adresu 16(%rbp), %eax przeności do rejestr %eax odpowiadającego
w ABI za wynik procedury to wartość pod adresem 16(%rbp) jest argumentem dla bar.
Jednocześnie pamiętamy, że pierwsze 6 argumentów zgodnie z ABI umieszczamy w rejestrach.
Jako że zwracana wartość ma 4 bajty, nie mamy pewności co znajduje się pod adresem A-12.
Stąd wiemy jedynie, że bar przyjmuje co najmniej n=7n=7 argumentów.

Przykładowe wywołanie procedury bar:

;; tutaj można dokładać kolejne argumenty, na przykład:
;; pushq (* 9-ty argument *)
;; pushq (* 8-my argument *)

pushl (* 7-my argument *)
subq $8, %rsp   

movl (* 1-szy argument *), %edi
movl (* 2-gi argument *), %esi
movl (* 3-ci argument *), %edx
movl (* 4-ty argument *), %ecx
movl (* 5-ty argument *), %r8d
movl (* 6-ty argument *), %r9d

call bar
